──────────────────────────────────────────────────────────────────────────────
STEP 4 — PICK YOUR AUTH PATH (this decides whether you need a refresh_token)
──────────────────────────────────────────────────────────────────────────────
Our backend supports BOTH. Enable whichever the portal offers; enable both if
unsure.

OPTION A — client_credentials  (simplest; NO refresh_token needed)
  Works when the app's own account is authorized for the plant data directly.
  - Enable the "Client Credentials" grant on the app.  [VERIFY ON SCREEN]
  - You're done with auth — our backend mints tokens from client_id + secret.
  - Skip straight to Step 5.

OPTION B — Authorization Code + refresh_token  (likely required if the plant
           owner account differs from the app, or SMA demands owner consent)
  - Enable the "Authorization Code" grant + refresh tokens (offline_access).  [VERIFY ON SCREEN]
  - Redirect URI on the app must match what you use below — use
        https://localhost/callback
    (or a redirect URI the engineer gives you).
  - Get the initial refresh_token (one-time owner consent):
    4b-1. Open this in a browser (replace YOUR_CLIENT_ID; confirm the authorize
          PATH with the engineer if it errors — the host auth.smaapis.de is
          correct, the /oauth2/authorize path is the standard one but UNVERIFIED):
            https://auth.smaapis.de/oauth2/authorize?response_type=code&client_id=YOUR_CLIENT_ID&redirect_uri=https://localhost/callback&scope=monitoring%20offline_access
    4b-2. Log in as the Tannery Brook OWNER and click Allow / Authorize.
    4b-3. The browser redirects to https://localhost/callback?code=XXXX . The
          page may fail to load — that's fine; copy the code=XXXX value from the
          address bar (it expires in minutes, so move quickly).
    4b-4. Exchange the code for tokens (terminal):
            curl -s -X POST https://auth.smaapis.de/oauth2/token \
              -u "YOUR_CLIENT_ID:YOUR_CLIENT_SECRET" \
              -d "grant_type=authorization_code" \
              -d "code=THE_CODE" \
              -d "redirect_uri=https://localhost/callback"
    4b-5. The JSON response includes "refresh_token" — COPY it. That's the value
          the backend stores. (It also returns an access_token for Step 5.)
  If any of 4b is confusing, DON'T keep retrying — send the engineer the
  client_id + redirect URI and they'll walk it live.

──────────────────────────────────────────────────────────────────────────────
STEP 5 — FIND system_id + VERIFY IT ACTUALLY WORKS  (do this before sending)
──────────────────────────────────────────────────────────────────────────────
5a. Get an access token (terminal):

    -- Option A (client_credentials): --
    curl -s -X POST https://auth.smaapis.de/oauth2/token \
      -u "YOUR_CLIENT_ID:YOUR_CLIENT_SECRET" \
      -d "grant_type=client_credentials"

    -- Option B (refresh_token): --
    curl -s -X POST https://auth.smaapis.de/oauth2/token \
      -u "YOUR_CLIENT_ID:YOUR_CLIENT_SECRET" \
      -d "grant_type=refresh_token" \
      -d "refresh_token=YOUR_REFRESH_TOKEN"

    Copy the "access_token" from the JSON.

5b. List the plants your app can see — THIS is how you find system_id without
    hunting through the portal:

    curl -s https://monitoring.smaapis.de/v1/plants \
      -H "Authorization: Bearer ACCESS_TOKEN"

    Find "Tannery Brook" in the response and read its id → that's system_id.
    (Alternatively the plant ID is visible in the ennexOS/Sunny Portal URL when
    you open the plant: https://ennexos.sunnyportal.com .)

5c. Confirm the exact plant endpoint our backend hits:

    curl -s https://monitoring.smaapis.de/v1/plants/YOUR_SYSTEM_ID \
      -H "Authorization: Bearer ACCESS_TOKEN"

    SUCCESS = JSON describing Tannery Brook.
    401 = bad credentials/token. 403 = app not approved / owner hasn't consented.
    404 = wrong system_id.

5d. (Optional) Confirm the live-power feed the hub reads every tick:

    curl -s "https://monitoring.smaapis.de/v1/plants/YOUR_SYSTEM_ID/measurements/sets/EnergyAndPowerPv/Recent" \
      -H "Authorization: Bearer ACCESS_TOKEN"
